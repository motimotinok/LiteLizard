# LiteLizard はじめての使い方

LiteLizard は、文章をAIに書かせるための道具ではなく、自分の文章がどう読まれそうかを段落単位で観測するためのデスクトップ執筆環境です。

このページでは、初めて使う人が最初の価値を確認できるように、インストール後に1つの段落を Reading Agent に読ませ、返ってきた読みを推敲判断に使うところまでを説明します。

## どんな場面で使うか

LiteLizard は、次のような文章で使いやすいように作っています。

- 小説やエッセイで、読者がどこで興味を持つか、どこで引っかかるかを見たい
- 書いた段落の意味や温度が、自分の意図とずれていないか確認したい
- 編集者、初見読者、感覚的な読者など、複数の視点から同じ段落を読みたい
- AIの提案をそのまま採用するのではなく、自分で直す・直さないを判断したい

反対に、すべての文章を自動で添削したり、品質を点数で保証したりする用途には向いていません。LiteLizard の分析結果は、採用すべき命令ではなく、書き手が判断するための材料です。

## 最初の流れ

```mermaid

  A["DMGを入れる"] --> B["作業フォルダを開く"]
```

## 1. インストールする

MVP 公開版は Apple Silicon Mac 向けです。動作確認済み環境は macOS Tahoe 26.5.1 です。その他のmacOSでは実機確認を行っていないため、動作保証外です。

1. [LiteLizard-latest-arm64.dmg](https://github.com/motimotinok/LiteLizard/releases/download/mvp-latest/LiteLizard-latest-arm64.dmg) をダウンロードします。
2. `.dmg` を開き、`LiteLizard.app` を `Applications` フォルダへ移動します。
3. 未署名アプリのため、初回は `LiteLizard.app` を右クリックまたは Control クリックし、`開く` を選びます。

Gatekeeper 警告への詳しい対処は [README](../README.md#macos-へのインストール) を確認してください。

## 2. 作業フォルダを選ぶ

初回起動後、LiteLizard は作業フォルダを求めます。既存の原稿フォルダを選んでも、新しくフォルダを作ってもかまいません。

選んだフォルダには、LiteLizard が次のデータを保存します。

- `.lzl` 文書
- `.litelizard/analysis/` 配下の分析結果

アプリ設定、APIキー、Reading Agent は Electron の `userData` 配下に保存されます。保存と外部送信の詳しい説明は [PRIVACY.md](../PRIVACY.md) を確認してください。

## 3. 新しい文書を書く

作業フォルダを開いたら、空状態の画面から `新しい文書を書く` を選びます。まずは短い段落で十分です。

例:

> 彼女は返事の代わりに、冷めたカップへ指を添えた。

LiteLizard は、段落ごとの読みを扱う道具です。最初から長い原稿を用意しなくても、1段落だけで読みの流れを確認できます。

## 4. provider を設定する

分析を実行するには、次のいずれかを設定します。

- OpenAI APIキー
- Anthropic APIキー
- Local LLM endpoint とモデル名

左側の歯車アイコンから設定画面を開き、使う provider を保存してください。OpenAI / Anthropic を使う場合、分析実行時に対象段落、参照本文、Reading Agent の設定が外部 provider に送信されます。Local LLM を使う場合も、送信先は設定した endpoint に従います。

LiteLizard 自体には独自の分析サーバーや telemetry はありません。

## 5. Reading Agent を選ぶ

Reading Agent は、文章をどの視点で読むかを決める設定です。最初はテンプレートから1つ追加すると試しやすいです。

代表例:

- 初見の読者: 予備知識なしで、理解、期待、興味、引っかかりを見る
- 感覚を読む読者: 感情、身体感覚、空気、余韻を見る
- 構造編集者: 主題、構造、論理、重複、削除候補を見る
- 書き続ける伴走者: 勢い、発見、残してよい部分を見る

同じ段落でも、Agent を変えると返ってくる読みが変わります。正しい読み手を1つ選ぶというより、必要な観測器を切り替える感覚で使います。

## 6. 段落を読ませる

本文の段落を選ぶと、分析パネルでその段落に対する読みを扱えます。`段落を読ませる` を押すと、選択中の Reading Agent がその段落を読みます。

必要なときは `今回だけの観点` に、今回だけ見たいことを書けます。

例:

- 初見読者として引っかかる場所だけ見る
- この段落が前後の流れから浮いていないか見る
- 削るならどこかだけを見る

この追加観点は保存済みの Reading Agent には混ざりません。その回の分析だけに使われます。

## 7. 分析結果を判断に使う

分析結果は、自動で本文へ反映されません。返ってきた読みを見て、書き手が判断します。

例:

| 返ってきた読み | 判断例 |
|----------------|--------|
| 初見の読者が状況を理解できていない | 前段落に前提を足す、またはこの段落で状況を一語だけ補う |
| 感覚を読む読者が余韻を強く拾っている | その余韻を残したいなら直さない。テンポを上げたいなら次段落を短くする |
| 構造編集者が重複を指摘している | 同じ役割の文を削る。重複がリズムとして必要なら残す |
| 書き続ける伴走者が強い表現を拾っている | その表現を軸に次の段落を伸ばす |

大事なのは、AIの言う通りに直すことではありません。自分の文章が別の視点からどう見えたかを確認し、その読みを採用するかどうかを自分で決めることです。

## よくある最初の使い方

### エッセイの冒頭を見る

冒頭の1〜3段落を `初見の読者` に読ませます。読者が何を期待したか、何に興味を持ったか、どこで迷ったかを見ます。最初に直すべきなのは、文章全体ではなく、読み始めの足場かもしれません。

### 小説の空気を見る

印象を大事にしたい段落を `感覚を読む読者` に読ませます。意味ではなく、速度、温度、距離、余韻がどう届いたかを見ます。説明を足すべきか、あえて説明しないほうがよいかを判断しやすくなります。

### 構成の停滞を見る

話が進まないと感じる段落を `構造編集者` に読ませます。重複、論理の飛躍、役割の薄い文がないかを見ます。削除や移動の候補を得ても、文章の声や狙いを壊すなら採用しない選択もできます。

## 注意

- 分析結果は正解ではありません。
- 外部 provider を使う場合、分析対象の本文と参照本文はその provider へ送信されます。
- 現行の公開版は Apple Silicon Mac 向け MVP / beta です。
- 未署名アプリのため、初回起動時に macOS の警告が出る場合があります。
