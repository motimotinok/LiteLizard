# Update NOW

`NOW.md` を、LiteLizard の現在地を示す短い入口として更新する。

## 成果物

- `NOW.md`

必要がなければ他のファイルは更新しない。

## 成功条件

- `NOW.md` が 40 行以内を目安に保たれている
- 現在の主目的、優先判断、代表的な親 Issue、完了履歴の参照先だけが分かる
- GitHub Issue 一覧、詳細ロードマップ、細かい進捗、完了履歴を持っていない
- 公開方針や開発の重心が変わった時だけ更新されている

## 制約

- `NOW.md` をタスク台帳にしない
- open Issue を全件列挙しない
- `CHANGELOG.md` の代わりに完了履歴を書かない
- 退役済み Product Map を更新しない

## 更新方針

- 主目的が変わったら「現在の主目的」を 1〜2 文で更新する
- 人間が見るべき親 Issue や判断テーマが変わったら「優先判断」を更新する
- 参照先の役割が変わった時だけ「参照先」を更新する
- 細かい候補の整理が必要な場合は、`triage-issue-backlog` を使う

## 完了時の報告

更新した見出しと、`NOW.md` に残した理由を短く報告する。
