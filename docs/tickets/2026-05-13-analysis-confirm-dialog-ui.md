# 分析実行前確認ダイアログの表示崩れと文言を整える

## 背景

MVP 公開前の実 API 手動確認中、分析ボタンを押した後に表示されるコンテキスト消費予測の確認 UI で、予測値は表示されるものの「実行する」ボタンが通常操作画面から見えず、キャンセルしか選べないように見える状態が観測された。

同じ画面には「コンテキスト本文量」など内部実装寄りで、通常ユーザーが何を確認すればよいか分かりづらい文言も残っている。API キー登録後の最初の分析実行で詰まらないように、表示崩れと文言をまとめて整える。

## ゴール

分析実行前の確認 UI で、ユーザーが送信対象と概算量を理解し、「実行する / キャンセル」を迷わず選べる。

## スコープ

- 分析実行前のコンテキスト消費予測 / 確認 UI のレイアウト修正
- 実行ボタンとキャンセルボタンの表示、配置、重なり、スクロール、狭い画面幅での操作性確認
- ラベル、説明文、補足文、ボタン周辺の日本語文言の見直し
- 必要に応じた renderer test / snapshot 相当のテスト追加または更新
- Electron アプリ上での分析前確認 UI の手動確認

## 非ゴール

- 分析 provider の通信仕様変更
- コンテキスト予測ロジックの計算方法変更
- UI 全体のリデザイン
- 詳細なチュートリアルやヘルプページの追加
- 設定画面やエディタ画面など、確認 UI と無関係な改修

## 受け入れ条件

- [ ] 分析ボタン押下後の確認 UI で、実行ボタンとキャンセルボタンが両方見える
- [ ] 実行ボタンが他の要素に隠れたり、画面外に押し出されたりしない
- [ ] 実行ボタンから従来通り分析通信を開始できる
- [ ] キャンセルボタンから確認 UI を閉じられる
- [ ] 小さめのウィンドウ幅でも操作不能にならない
- [ ] 「コンテキスト群」など、ユーザーに意味が伝わりにくい内部用語が必要に応じて置き換えられている
- [ ] 分析前に何を確認すればよいかが分かる短く自然な日本語文言になっている
- [ ] API 送信や消費見込みに関する説明が README / PRIVACY.md の説明と矛盾していない
- [ ] 不要な大規模リファクタリングが含まれていない

## 検証方法

- [ ] 関連する既存テストを確認する
- [ ] 必要な renderer / store / static render test を追加または更新する
- [ ] Electron アプリ上で API キー設定後に分析ボタンを押し、確認 UI から実行 / キャンセルの両方を手動確認する
- [ ] `pnpm -w lint`
- [ ] `pnpm -w test`
- [ ] `pnpm -w build`
