# 分析実行前確認ダイアログの表示崩れと文言を整える

## 背景

MVP 公開前の実 API 手動確認中、分析ボタンを押した後に表示されるコンテキスト消費予測の確認 UI で、予測値は表示されるものの「実行する」ボタンが通常操作画面から見えず、キャンセルしか選べないように見える状態が観測された。

同じ画面には「コンテキスト本文量」など内部実装寄りで、通常ユーザーが何を確認すればよいか分かりづらい文言も残っている。API キー登録後の最初の分析実行で詰まらないように、表示崩れと文言をまとめて整える。

## ゴール

分析実行前の確認 UI で、ユーザーが送信対象と概算量を理解し、「実行する / キャンセル」を迷わず選べる。

## スコープ

- 分析実行前のコンテキスト消費予測 / 確認 UI のレイアウト修正
- 実行ボタンとキャンセルボタンの表示、配置、重なり、スクロール、狭い画面幅での操作性確認
- ラベル、説明文、補足文、ボタン周辺の日本語文言の見直し
- 必要に応じた renderer test / snapshot 相当のテスト追加または更新
- Electron アプリ上での分析前確認 UI の手動確認

## 非ゴール

- 分析 provider の通信仕様変更
- コンテキスト予測ロジックの計算方法変更
- UI 全体のリデザイン
- 詳細なチュートリアルやヘルプページの追加
- 設定画面やエディタ画面など、確認 UI と無関係な改修

## 完了メモ

- 確認ダイアログを `apps/desktop/src/renderer/components/AnalysisRunConfirm.tsx` に props 駆動で切り出し、`AnalysisPane` の `.analysis-header` 内 flex 子要素ではなく `.analysis-shell` 直下の overlay として描画するよう変更した。`.analysis-shell { overflow: hidden }` の影響で確認カードがクリップされて実行ボタンが画面外に押し出される問題は、これで構造的に解消した。
- overlay には半透明 backdrop（クリックでキャンセル）と最大高制限内スクロール、`width: min(100%, 320px)` のカード幅を与え、小さなウィンドウ幅でも操作不能にならないようにした。
- 文言は内部用語を排除し「解析する段落 / 段落本文 / 前後の文脈 / 送信量(概算) / 応答量(概算)」に統一。リード文（送信対象の説明）と注意書き（概算であって課金額そのものではない旨）を追加した。
- `AnalysisRunConfirm.test.tsx` を新規追加し、SSR で実行/キャンセル両ボタン、`role="dialog"`、overlay 構造、内部実装寄りの旧文言が含まれないこと、日本語ロケールでの数値表示を 4 ケースで固定した。
- 手動確認: Electron 実機での GUI 動作確認はこのループでは実施せず、`docs/tickets/2026-05-13-mvp-release-verification-checklist.md` 側の手動チェック対象として委ねる。
- 検証: `pnpm -w lint` 成功、`pnpm -w test`（desktop 277 件、shared 57 件、api 4 件、e2e 6 skipped）成功、`pnpm -w build` 成功。
