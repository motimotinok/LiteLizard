# 全体解析の成功・失敗件数を表示する

## 背景

WBS R-08 では、全体解析の成功/失敗件数表示が未完了になっている。現在の解析は段落ごとに進捗を返し、成功した段落は UI に反映されるが、全体実行後に何件成功し、何件失敗したかをユーザーが一目で確認しづらい。

## ゴール

全体解析の完了後、対象段落数、成功件数、失敗件数が UI またはステータスメッセージで分かる。

## スコープ

- `runAnalysis` / `onAnalysisProgress` 周辺の現在の成功・失敗扱いを確認する
- 全体解析の対象件数、成功件数、失敗件数を renderer state で扱う
- AnalysisPane または既存のステータスメッセージに、静かな UI トーンで件数を表示する
- 一部失敗時でも成功済み段落の結果が残ることを確認する
- 必要な store / component テストを追加または更新する

## 非ゴール

- 解析リトライ UI の本格実装
- 解析履歴ファイル形式の変更
- 章サマリー解析や分析モードの追加
- 解析パネル全体の再設計

## 受け入れ条件

- [x] 全体解析の実行後に成功件数と失敗件数が分かる
- [x] 解析対象が 0 件の場合も表示や状態が破綻しない
- [x] 一部失敗時に成功済み結果が失われない
- [x] 既存の段落単位再解析と Reading Agent 適用が壊れていない

## 検証方法

- [x] 関連する既存テストを確認する
- [x] 必要なテストを追加または更新する
- [x] `pnpm -w lint`
- [x] `pnpm -w test`
- [x] `pnpm -w build`

## 完了メモ

`analysisRunSummary` を renderer store に追加し、全体解析の対象/成功/失敗件数を status message と AnalysisPane に表示するようにした。結果が返らなかった対象段落は failed にして、一部成功した解析結果は保持する。
