# 現在の文書内検索画面を実装する

## 背景

WBS R-20 では、検索画面の中身が未完了になっている。左メニューには検索ボタンがあるが、まずはプロジェクト全体検索ではなく、現在開いている文書内のテキスト検索だけを対象にする。

## ゴール

検索画面で現在の文書内の本文を検索し、該当段落へ移動または選択できる。

## スコープ

- LeftIconRail の検索ボタンと workspace panel 切替を確認する
- 現在の文書の title / chapter / paragraph text を対象に検索する
- 検索結果を段落単位で表示し、クリックで対象段落を active にする
- 検索語が空、該当なし、文書未選択の場合の表示を整える
- 必要な store / component テストを追加または更新する

## 非ゴール

- プロジェクト全体検索
- ファイル名検索
- 正規表現検索
- 置換機能
- インデックス作成や全文検索エンジン導入

## 受け入れ条件

- [ ] 左メニューの検索ボタンから検索画面を開ける
- [ ] 現在の文書内の段落本文を検索できる
- [ ] 検索結果から対象段落へ移動または active 選択できる
- [ ] 該当なし、文書未選択、空検索で表示が破綻しない
- [ ] 既存のエディター、分析パネル、設定画面への遷移が壊れていない

## 検証方法

- [ ] 関連する既存テストを確認する
- [ ] 必要なテストを追加または更新する
- [ ] `pnpm -w lint`
- [ ] `pnpm -w test`
- [ ] `pnpm -w build`
