# 縦書き執筆モード方針

関連Issue: #131  
状態: decision（現行公開版では実装しない）

## 結論

LiteLizard は現行公開版では、縦書きに対応した執筆専用モードを実装しない。

縦書きは日本語執筆体験として価値があるが、現行の Lexical エディタ、段落 DnD、章/段落ガター、分析レーン、Undo/Redo、スクロール連動は横書き・縦方向リストを前提にしている。CSS の `writing-mode` 追加だけで実用的な日本語編集体験が成立するとみなさない。

## 非採用理由

- Lexical 上の IME 入力、カーソル移動、選択、コピー&ペースト、Undo/Redo を縦書きで実機確認する必要がある。
- dnd-kit の段落/章並び替えは現在 `verticalListSortingStrategy` を使っており、縦書き時の左右方向操作へ単純には移せない。
- 段落番号、ドラッグハンドル、章レール、本文ハイライトは左余白を前提にしており、縦書きでは配置意味が変わる。
- 分析モードは本文左・分析右の横並びと段落連動を前提にしており、縦書き執筆モードと同時に扱うと本文領域が圧迫される。
- 保守停止前提では、縦書き特有のブラウザ/Electron/IME差分を継続検証できない。

## 将来再検討する場合の必須検証

実装前に、Electron 上で次を確認する。

- 日本語 IME 入力、変換、確定、再変換
- カーソル移動、文字選択、コピー&ペースト
- Enter による段落分割、Backspace による段落統合
- Undo / Redo
- 長文スクロールと編集位置復元
- 章/段落構造ガターの表示位置
- DnD 並び替え方向とキーボード操作
- 分析モードへ戻したときの段落対応と分析履歴維持

## 採用する場合の構成案

将来実装するなら、縦書きは分析モードの一表示オプションではなく、執筆専用モードとして扱う。

- 執筆モード: サイドバー、分析パネル、段落読みレーンを隠し、本文編集に集中する
- 分析モード: 現行の横書き本文 + 右インスペクターを維持する
- 書字方向は文書データへ保存せず、アプリ表示設定として扱う
- 縦書き中も本文構造 `chapterId` / `paragraphId` / `order` は変えない

## 現行仕様との関係

`docs/LiteLizard_spec_v003.md` には縦書き/横書き切り替えの将来案が残っているが、これは現行実装の約束ではない。現行公開版の正は、この文書の「実装しない」判断とする。
