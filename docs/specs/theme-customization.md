# 表示カスタマイズ方針

関連Issue: #114  
状態: decision（自由な画面配色・フォントカスタマイズは採用しない）

## 結論

LiteLizard は、画面全体の配色・アクセント色・任意フォントを自由にカスタマイズする機能を当面実装しない。既存の Editor Tweaks を、公開版で支える表示調整の範囲とする。

現在支える範囲:

- 本文書体: 明朝 / ゴシック
- 本文サイズ
- 行間
- 紙面の黄ばみ強度
- 分析パネルの横並び / オーバーレイ

これ以上の自由な色指定、任意フォント入力、OSフォント列挙、テーマ保存/共有は実装しない。

## 非採用理由

- LiteLizard のデザインは、静かな執筆環境としての可読性、低ノイズ、macOS になじむ補助UIを重視している。
- 背景色、本文色、アクセント色を自由入力にすると、本文の読みやすさ、ボタン識別、エラー/警告表示、分析タグ表示のコントラスト保証が難しくなる。
- 任意フォントは、OS差異、ライセンス、表示崩れ、行高計算、縦横メトリクス差分を持ち込みやすい。
- 保守停止前提では、テーマプリセットやカラーピッカーの品質・アクセシビリティ・回帰確認を継続できない。
- 文章の見え方を変えて検査する用途は有効だが、#115 の読書環境プレビューと同様、現時点では主要導線に入れない。

## 既存 Editor Tweaks を正とする理由

既存の Editor Tweaks は、自由度を限定しつつ、執筆体験に直接効く要素だけを扱う。

- 書体は同梱済みフォントと CSS 変数の範囲に閉じる
- 本文サイズと行間は読みやすい範囲に clamp される
- 紙面色は単一の暖かさパラメータから算出し、コントラストを大きく崩さない
- 分析パネル配置は本文の圧迫感を調整するための実用設定である

この範囲なら、ユーザーが自分の書き味を整えつつ、UI全体の可読性と操作性を壊しにくい。

## 将来再検討する条件

自由テーマを再検討する場合は、先に次を満たす。

- 色プリセットごとのコントラスト検証方法がある
- エラー、警告、成功、選択状態、分析タグの色規則を壊さない
- 任意フォントではなく、同梱または明示的に検証済みの候補に限定する
- 初期値へ戻す操作があり、設定ファイル破損時も安全に既定値へ戻る
- 執筆画面と補助画面の両方で表示確認できる検証経路がある

この条件を満たさない限り、表示調整は Editor Tweaks の範囲に留める。
