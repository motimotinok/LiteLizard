# ID 生成・重複検出仕様

関連タスク: S-03
決定経緯: `docs/decisions.md` [2026-03-20] S-03

---

## 1. ID フォーマット

全レイヤーで統一する。

| 種類 | プレフィックス | パターン | 例 |
|------|---------------|----------|-----|
| 文書 | `d_` | `d_[a-z0-9]{10}` | `d_a8f3k2m9x1` |
| 章 | `c_` | `c_[a-z0-9]{10}` | `c_x1y2z3w4a5` |
| 段落 | `p_` | `p_[a-z0-9]{10}` | `p_abc123def4` |

## 2. ID 生成アルゴリズム

```typescript
function generateId(prefix: 'd' | 'c' | 'p'): string {
  return `${prefix}_${randomAlphanumeric(10)}`;
}

function randomAlphanumeric(length: number): string {
  // [a-z0-9] からランダムに length 文字生成
  // Math.random().toString(36) ベース
}
```

- 衝突確率: 36^10 ≈ 3.6 × 10^15（実用上無視できる）
- 外部ライブラリ不要
- **生成関数は1箇所に集約**し、全レイヤーで共有する

---

## 3. ID の永続性

- `paragraphId` はテキスト編集・並び替えで変わらない
- `chapterId` は章の追加・削除・並び替えで変わらない
- `documentId` はファイルのリネームで変わらない
- ID が変わるのは: 破損時の自動修復、ファイルコピー時の重複解消のみ

---

## 4. 重複検出

### 4.1 検出タイミング

**プロジェクト起動時**に全 `.lzl` ファイルのフロントマター（先頭数行）を読み、`Map<documentId, filePath>` をメモリ上に構築する。

- フロントマターの先頭数行を読むだけなので、100ファイルでも数十ms
- マップはメモリ上に保持し、ファイル追加/削除時に更新
- ファイルオープン時はこのマップと照合

### 4.2 重複解消

- **後から開いた方**に自動で新 ID を付与する
- サイレント処理（ユーザー通知なし）
- コンソールログのみ出力（デバッグ用）
- 確認ダイアログは不要（重複はファイルコピー時にのみ発生し、ユーザーの意図的操作ではない）

### 4.3 同一ファイル内の重複

パース時に同一ファイル内で同じ ID が2回以上出現した場合、2つ目以降に新 ID を振る（`lzl-format.md` §9.1 チェック6）。
