# 章削除・段落統合エッジケース仕様

関連タスク: S-07
決定経緯: `docs/decisions.md` [2026-04-02] S-07

---

## 1. 章削除

章を削除すると、その章に含まれる段落は**前の章の末尾に移動**される。

### 1.1 通常ケース（前の章が存在する）

- 削除対象の章に含まれるすべての段落を、前の章の段落リスト末尾に追加する
- `order` は前の章の既存段落の最大値から連番で振り直す

```
Before:
  章A: [p1(order:1), p2(order:2)]
  章B: [p3(order:1), p4(order:2)]  ← 削除

After:
  章A: [p1(order:1), p2(order:2), p3(order:3), p4(order:4)]
```

### 1.2 先頭の章を削除

前の章が存在しないため、段落を受け入れる先がない。

- 削除対象の段落を保持したまま、**新しい「無題の章」を生成**する
- 新しい章の `title` は空文字列 `""`
- 段落の `order` はそのまま維持

### 1.3 最後の1章を削除（文書内に章が1つだけ）

1.2 と同じ挙動。段落を残して「無題の章」を生成する。

- 文書に章が0個の状態は発生しない（常に最低1章が存在する）

### 1.4 空章の削除

段落が0個の章は削除可能。移動する段落がないため、章を単純に削除する。

### 1.5 確認ダイアログ

**出さない。** 将来の Undo/Redo 実装（S-08）で復元可能にすることで担保する。

## 2. 段落統合（Backspace）

段落の先頭にカーソルがある状態で Backspace を押すと、**前の段落のテキスト末尾にカレント段落のテキストを結合**する。

### 2.1 通常ケース

```
Before:
  段落A: "吾輩は猫である。"
  段落B: "名前はまだ無い。"  ← 先頭で Backspace

After:
  段落A: "吾輩は猫である。名前はまだ無い。"
  （段落B は消滅）
```

- カーソルは結合後のテキスト内で、元の段落Aテキスト末尾の位置に配置する
- 段落B の分析結果は破棄される（段落が消滅するため）
- 後続の段落の `order` を振り直す

### 2.2 章をまたぐ統合はしない

章の最初の段落の先頭で Backspace を押しても**何も起きない**。

- 前の章の最後の段落との結合は行わない
- 章境界は Backspace では越えられない

---

## 3. 空章の扱い

- 段落が0個の章は**許容する**（自動削除しない）
- 空章はユーザーが明示的に削除するまで残る
- DnD で全段落を別の章に移動した結果、空章になるケースが想定される

---

## 4. エッジケースまとめ

| 状況 | 挙動 |
|------|------|
| 章削除（前の章あり） | 段落を前の章末尾に移動 |
| 先頭の章を削除 | 段落を残して「無題の章」を生成 |
| 最後の1章を削除 | 同上 |
| 空章を削除 | 単純に章を削除 |
| 段落先頭 Backspace（同一章内） | 前の段落とテキスト結合 |
| 段落先頭 Backspace（章の最初の段落） | 何も起きない |
| 単一段落章の先頭 Backspace | 何も起きない（章をまたがない） |
| 空章（段落0個） | 許容する（自動削除しない） |
