# 抽象化した読書環境プレビュー方針

関連Issue: #115  
状態: decision（当面は実装しない）

## 結論

LiteLizard は当面、原稿を複数の抽象化された読書環境プリセットへ置き直すプレビュー機能を実装しない。

文章の見え方を変えて推敲する発想自体は有効だが、現行の優先度では、恒常的なテーマ/フォント基盤を広げる #114 を先に実装しない限り、読書環境プレビューだけを独立して持つ価値より UI と保存契約の複雑化が上回る。

## 非採用理由

- #115 は #114 の表示設定基盤に依存しているが、現時点では #114 も公開後の継続改善前提の重い機能である。
- LiteLizard の中核は本文編集と Reading Agent による読まれ方の観測であり、複数レイアウトの見た目比較は補助的価値に留まる。
- 読書環境プリセットを入れると、恒常設定、一時プレビュー、文書保存、分析表示、執筆モードの関係を新たに管理する必要がある。
- 特定サービスを模倣しない抽象プリセットは安全だが、価値を伝えるには説明と UI 導線が必要になり、静かな執筆画面にノイズを増やしやすい。
- 保守停止前提では、表示プリセットの品質調整やユーザー環境差分への追従を続けにくい。

## 将来再検討する場合の境界

再検討する場合も、次を守る。

- 特定サービス名、ロゴ、固有UIを模倣しない
- プレビュー状態を `.lzl` 文書へ保存しない
- ユーザーの恒常テーマ設定を変更しない
- 本文、分析結果、Reading Agent、履歴を変更しない
- 通常編集表示へワンアクションで戻れる
- プレビュー中であることと、実際の掲載結果を保証しないことを明示する

## プリセット候補

将来の検討用に、固有サービスへ依存しない候補だけを残す。

| プリセット | 確認したいこと |
|---|---|
| 長文記事 | 本文幅が広めの環境で、段落の長さ、説明過多、見出しの間隔を確認する |
| 短文タイムライン | 狭い幅と短い可視範囲で、冒頭の引き、1文の重さ、改行リズムを確認する |
| 無装飾テキスト | 装飾なしで、語順、重複、論理の流れだけを確認する |
| モバイル読書 | 小さな幅で、段落密度、スクロール量、文の切れ目を確認する |

## #114 との関係

この機能は #114 の恒常テーマ設定とは別物として扱う。#114 が実装される場合でも、テーマ設定はユーザーの普段の執筆環境を整えるためのもの、読書環境プレビューは一時的に見え方を変えて推敲するためのものとして分離する。

ただし、現時点では #114 も #115 も実装しない。表示調整は既存のエディタ調整（書体、本文サイズ、行間、黄ばみ、分析パネル配置）に留める。
