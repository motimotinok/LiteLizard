## [2026-03-14] 分析機能の役割・製品構造を再定義

- **決定**: 分析ペインの役割を「読者の感想・評価」ではなく「書き手の意図が読者にどう届いたか（意図と解釈の擦り合わせ）」として定義する
- **理由**: 「面白かった」系の評価フィードバックは問題指摘マシンになりやすく、執筆体験を損なう。「この段落はこういう意図として読み取られる」という形式にすることで、書き手が自分で判断できる情報を渡せる
- **製品構造の整理**:
  - コア機能 = 書きやすさの追求（縦書き・横書き対応、執筆体験の洗練）
  - 分析 = 完成後レビューとして位置づけ（執筆中のリアルタイム分析ではない）
  - 分析機能はサブスク限定の付加価値として設計する
- **将来実装（MVPスコープ外）**:
  - 書き手の「意図宣言」フィールド（段落ごとに意図を記述）
  - 意図 vs 読者解釈のギャップ可視化
  - 感情アークのチャート表示（段落をまたいだ emotion タグの流れ）
- **却下した方向性**: 「問題点を指摘して書き直しを促すアシスタント」→ 書き手の主導権を損なうため採用しない
- **この決定が現在の実装に与える影響**: deepMeaning の文体方針（読者体感 + 書き手への気づきの混在）はこの定義と整合している

## [2026-03-20] .lzl 内部フォーマット: フロントマター + インラインマーカー方式を採用 (S-01)

- **決定**: .lzl ファイルを YAML フロントマター + HTML コメント形式のインラインマーカー（`<!--:: ch/p {id} ::-->`）で構造化する。フォーマットバージョン `lzl-v1`
- **仕様**: `docs/specs/lzl-format.md`
- **理由**:
  - ID と本文が物理的に隣接 → 部分破損時の被害が局所化される（破損耐性の最優先原則）
  - 1ファイルで構造が完結する（自己記述性）。外部メタファイルに依存しない
  - 空行 = 段落境界ルールにより、マーカーが消失しても段落境界を推定可能
  - フロントマターに章・段落カウントを冗長データとして持ち、パース後に破損検出が可能
  - 章タイトルはマーカー内に1行完結で格納（`## 見出し` との誤認を防ぐ）
- **却下した案**:
  - ヘッダー集約方式: 本文と構造の物理的距離が大きく、部分破損時に構造全体が影響を受ける
  - ブロック区切り方式: 冗長性は高いが構文が重く、パーサーの複雑化に見合わない
  - メタデータ分離方式（.lzl + .lzl.meta）: 2ファイル同期の破綻リスクが破損耐性と矛盾。メタファイル喪失で全構造消失
- **前提条件（この前提が崩れたら再検討）**:
  - Git 管理しない
  - 段落内改行を許容しない
  - 外部エディタでの編集を想定しない（閲覧のみ）

## [2026-03-20] documentId 生成・重複検出の仕様決定 (S-03)

- **決定**: `d_/c_/p_` + `randomAlphanumeric(10)` に統一。プロジェクト起動時に全スキャンして重複検出、サイレントで再採番
- **仕様**: `docs/specs/document-id.md`
- **理由**: 外部ライブラリ不要（衝突確率 36^10 で十分）。`d_` は `{prefix}_` 形式で将来の衝突なし。重複はコピー時のみ発生するため通知不要
- **却下した案**: ファイルオープン時のみ検出（比較マップがない）、確認ダイアログ（ユーザー判断不要）

## [2026-03-21] IPC チャンネル定義と型安全ブリッジ設計 (E-01)

- **決定**: `BridgeApi` インターフェースと `IPC_CHANNELS` 定数マップを `@litelizard/shared` に配置し、main / preload / renderer の3層で共有する。preload は `ipcRenderer.invoke` による実 IPC ブリッジに切り替え
- **理由**: API 契約の Single Source of Truth を確立し、チャンネル名の typo を型レベルで防止する。ブラウザ dev モードでは `main.tsx` のモック fallback で動作するため、preload からモック依存を除去できる
- **却下した案**:
  - `apps/desktop/src/shared/` に配置: preload の tsconfig（rootDir=src/preload）からインポートできない
  - パラメータ/戻り値の完全型チェック付き registerHandler ラッパー: IPC 境界はシリアライズされるため実行時は any。過剰な型安全は誤った保証になる
- **参照**: `packages/shared/src/bridge.ts`, `apps/desktop/src/preload/ipcBridge.ts`
