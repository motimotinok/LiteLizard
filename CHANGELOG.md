# CHANGELOG

プロジェクトの変遷を時系列で記録するファイル。
LLM へのコンテキスト提供・タスク生成の補助を主目的とする。

**記録ルール:**
- タスク完了のたびに追記（新しいエントリを末尾に追加）
- 設計判断・変更の意図を重視する（コードの詳細より「なぜ」を優先）
- 残課題があれば必ず書く

---

## テンプレート

```
## YYYY-MM-DD: {タスク名 or 変更の一行タイトル}

**変更内容**:
{何を実装・変更したか（設計判断・構成変更の概要）}

**背景・動機**:
{なぜこの変更が必要だったか}

**残課題・次のステップ**:
{未解決の問題、次に繋がるヒント。なければ「なし」}

---
```
