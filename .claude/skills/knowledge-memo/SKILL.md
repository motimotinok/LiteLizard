# Knowledge Memo — ナレッジ記録スキル

開発の中で気づいた知見・問題点・改善策を `PROJECTMEMORY/ナレッジ.md` に書き出すスキル。

このスキルが存在する理由: 作業中に気づいた「次回こうすれば良かった」「この方法には問題があった」という学びは、その場で記録しないと消える。ナレッジ.md に蓄積することで、次のセッションや開発サイクルで同じ失敗を繰り返さず、良い方法を再発見するコストをゼロにする。

---

## 記録する内容

以下のような気づきが対象:

- **開発フローの改善点** — 「このタスクの順番は逆の方が効率的だった」「このステップを自動化できる」
- **既存の方法の問題点** — 「この構成だとXXXの場面で困る」「このスキルの使い方がノイズになっている」
- **解決策・改善策** — 問題点とセットで、どう改善すべきかの案
- **ツール・ワークフローの知見** — Claude Code / git / その他ツールの効果的な使い方・落とし穴

コードの実装詳細や特定バグの修正手順は対象外（それは git の commit メッセージや decisions.md に入れる）。

---

## ナレッジ.md のフォーマット

```markdown
## YYYY-MM-DD: {一行タイトル}

**カテゴリ**: フロー改善 / 問題点 / ツール知見 / その他

**気づき**:
{何が問題だったか、または何が良かったか}

**改善策・対応**:
{次回どうするか、または今後の方針}

---
```

---

## 記録の手順

1. `PROJECTMEMORY/ナレッジ.md` を読む（既存エントリとの重複確認）
2. ユーザーの発話から気づきの内容を整理する
3. フォーマットに沿ってエントリを追記する（**新しいエントリはファイルの末尾に追加**）
4. 追記した内容を一行でユーザーに報告する

---

## 注意事項

- **既存エントリは削除・編集しない**。追記のみ行う
- タイトルは「何についての知見か」が一目で分かる表現にする
- 改善策がまだない場合は「要検討」と書いてよい。記録すること自体に価値がある
