# Update WBS Changelog

完了したタスクを `docs/wbs.md` と `CHANGELOG.md` に反映する。

このスキルの目的は、タスク台帳と変更履歴を常に同期させ、完了報告のたびに更新漏れが出ない状態を保つこと。

## 更新対象

- `docs/wbs.md`
- `CHANGELOG.md`

上記2ファイルを1セットで扱う。片方だけ更新して終わらない。

## 実行手順

### 1. 現状を確認する

1. `docs/wbs.md` を読む
2. `CHANGELOG.md` を読む
3. ユーザーの依頼または直前の作業内容から、完了したタスク ID と短い要約を特定する

タスク ID、完了扱いにする範囲、要約文のいずれかが曖昧なときだけ確認する。

### 2. `docs/wbs.md` を更新する

以下を最小差分で反映する。

- 対象タスクの状態を `✅` に更新する
- WBS 冒頭の `最終更新` がある場合は当日の日付に更新する
- 同じタスクについて詳細セクションに明示的な完了欄や完了メモがすでにある場合だけ、それも整合するよう更新する

やりすぎないこと:

- 関係ないタスクの並び替えをしない
- 担当、ブランチ、優先度を勝手に再設計しない
- `TASKS.md` や他の台帳に同じ内容を転記しない

### 3. `CHANGELOG.md` を更新する

新しい履歴はファイルの上側に入れる。古い履歴を下へ送る。

追記フォーマットは毎回この2行に統一する。

```text
[YYYY/MM/DD]短いタスク名（要約）
実装した内容の簡単なまとめ
```

ルール:

- 1タスクにつき2行だけ書く
- 1行目は日付 + 短いタスク名
- 2行目は何を実装したかの簡潔な要約にする
- 新しいエントリを既存の履歴より前に入れる
- 既存ファイルに古いテンプレートや説明文が残っていても、それには従わず、新しい履歴を先頭側に積む

`# CHANGELOG` の見出しがある場合は、その直下に最新エントリを入れる。見出しがない場合はファイル先頭に入れる。

### 4. 仕上げを確認する

更新後に以下を確認する。

- WBS の対象タスクが `✅` になっている
- `CHANGELOG.md` の先頭側に今回の2行が追加されている
- どちらのファイルにも不要な大規模整形が入っていない

## 書き方の基準

- WBS は台帳として正確さを優先する
- CHANGELOG は未来の自分や別エージェントが一目で流れを追える短さを優先する
- CHANGELOG の2行目は実装詳細の列挙ではなく、変更の中身が分かる自然文にする

良い例:

```text
[2026/04/18]R-02 Backspace段落統合
同一章内で段落先頭の Backspace 時に前段落へ統合されるよう実装した
```

避ける例:

```text
[2026/04/18]R-02
いろいろ修正
```

## 注意事項

- このスキルは完了反映用。新規タスク追加や優先度整理は主目的ではない
- `CHANGELOG.md` には新しい履歴を下へ追記しない
- ユーザーが明示しない限り、履歴の文体統一のためだけに過去ログを全面書き換えしない
